🧩

High-version support

One backend and a zero-NMS plugin cover Minecraft 1.13 through the latest release.

🔒

In-memory assets

Visuals load only in client memory, never touch disk, and are cleared the moment the player disconnects.

🛡️

Signed delivery

Assets travel over HTTPS with HMAC-signed URLs and are verified by SHA-256 content hash.

🎯

Selective replacement

A render-dispatcher mixin replaces only flagged entities; everything else renders vanilla.

An experiment in remote-delivered rendering

EntityReskin started from one question: can a Java Edition client fetch a model from a backend at runtime and use it to change how a live entity is rendered? The result is an end-to-end system — a shared binary protocol, a Spring Boot asset backend, a Bukkit relay plugin, and a Fabric client that bakes Bedrock-format assets in memory with GeckoLib.

Control and data planes

Identifiers travel over a plugin-messaging control channel; bulk assets travel over a separate HTTPS path.

Integrity end to end

A deterministic, content-addressed manifest pins assets by SHA-256; the client verifies every byte before it bakes.
